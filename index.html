Elementos de Páginas Web

  1. Orígenes de la Web.
    1. Web 2.0: Se refiere a la evolución de la web hacia una plataforma más interactiva, donde los usuarios generan contenido (blogs, redes sociales) y colaboran en línea. Se caracteriza por el uso de tecnologías como AJAX y APIs.
    2. Web 3.0 3: También conocida como la web semántica, busca hacer que los datos en la web sean más comprensibles para las máquinas. Utiliza tecnologías como RDF y ontologías, permitiendo conexiones más inteligentes entre la información.
    3. La W3C y los estándares: La World Wide Web Consortium (W3C) es la organización que establece estándares para la web. Su objetivo es promover la interoperabilidad y asegurar que la web sea accesible y utilizable para todos.
  2. Lenguajes de etiquetas.
    1. SGML: El Standard Generalized Markup Language es un lenguaje de marcado que sirve como base para otros lenguajes, incluyendo HTML y XML. Permite definir la estructura de documentos.
    2. XML: Extensible Markup Language es un lenguaje que permite almacenar y transportar datos. Es flexible y puede ser adaptado para diferentes aplicaciones.
    3. HTML: Hypertext Markup Language es el lenguaje de marcado utilizado para crear páginas web. Define la estructura del contenido en la web.
    4. XHTML: Extensible Hypertext Markup Language es una versión más estricta de HTML que combina la flexibilidad de XML. Busca asegurar que el código sea bien formado.
    5. HTML5: La última versión de HTML que incluye nuevas características como soporte multimedia nativo (audio y video), almacenamiento local y API mejoradas para una mayor interactividad.
  3. Otros lenguajes relacionados al desarrollo web.
    1. CSS/CSS3: Cascading Style Sheets es un lenguaje utilizado para describir la presentación de un documento HTML. CSS3 introdujo nuevas propiedades y características como transiciones y animaciones.
    2. JavaScript: Un lenguaje de programación que permite implementar funciones complejas en las páginas web, como interactividad y manipulación del DOM.
    3. Ajax: Asynchronous JavaScript and XML es una técnica que permite la actualización asíncrona de las páginas web, mejorando la experiencia del usuario al evitar recargas completas.
    4. Tecnologías y lenguajes del lado del servidor: Incluye lenguajes como PHP, Python y Ruby que se utilizan para crear la lógica de backend de las aplicaciones web, gestionando bases de datos y solicitudes del cliente.
    5. Navegadores: Aplicaciones que permiten a los usuarios acceder y visualizar contenido web. Ejemplos incluyen Chrome, Firefox y Safari.
    6. Motores renderizados: Son los componentes de los navegadores que convierten el HTML, CSS y JavaScript en una representación visual que los usuarios pueden ver y con la que pueden interactuar.
  4. Herramientas de desarrollo.
    1. Software WYSIWYG: "What You See Is What You Get" permite a los desarrolladores diseñar páginas web visualmente sin necesidad de escribir código manualmente.
    2. Otras alternativas
      1. Figma: Herramienta de diseño colaborativo que permite crear interfaces de usuario.
      2. InVisión: Plataforma para prototipado y colaboración en diseño.
      3. Zeplin: Herramienta que facilita la colaboración entre diseñadores y desarrolladores, generando especificaciones de diseño automáticamente.
      4. GitHub: Plataforma de control de versiones que permite a los desarrolladores colaborar y gestionar proyectos de software.
    3. Inspección de código desde el navegador: Herramienta que permite a los desarrolladores analizar y modificar el código HTML y CSS de una página web en tiempo real, ayudando en la depuración y optimización del sitio.

El Diseño De Una Página Web

  1. Tipos de páginas Web: Existen diversos tipos de páginas web, cada una diseñada para cumplir objetivos específicos. Las páginas estáticas, por ejemplo, presentan contenido fijo y son ideales para mostrar información básica, como portafolios o sitios de contacto. En contraste, las páginas dinámicas generan contenido en tiempo real y son comunes en blogs, plataformas de comercio electrónico y redes sociales. Otros tipos incluyen sitios web corporativos, que representan empresas y sus servicios, y páginas de destino, que se enfocan en convertir visitantes en clientes a través de acciones específicas.
  2. Puntos básicos en la elaboración de una página Web: La elaboración de una página web implica varios puntos clave Primero, es fundamental definir el objetivo del sitio y su público objetivo para orientar el contenido y el diseño. Luego, se debe planificar la estructura del sitio mediante un mapa que facilite la navegación. Un diseño visual atractivo y una experiencia de usuario intuitiva son esenciales para mantener el interés del visitante. Además, el contenido debe ser de alta calidad y optimizado para SEO, garantizando así una buena visibilidad en los motores de búsqueda. Finalmente, es crucial realizar pruebas en diferentes dispositivos y mantener el sitio actualizado para asegurar su relevancia y funcionalidad a largo plazo.

Estructura De Una Página Web

  1. Jerarquía de directorio de un proyecto web: se refiere a la organización estructurada de carpetas y archivos dentro del proyecto, diseñada para facilitar su desarrollo, mantenimiento y escalabilidad.
  2. Elementos de estructura para el diseño de una página Web:
    1. Regla utilizada para validar y certificar una página web: La validación y certificación de una página web se basa en el cumplimiento de las normas establecidas por el W3C (World Wide Web Consortium). Estas normas aseguran que el código HTML utilizado sea correcto, accesible y optimizado para diferentes navegadores y dispositivos. Un validador de HTML evalúa el documento, verificando aspectos como la correcta anidación de etiquetas, la declaración adecuada del tipo de documento (DOCTYPE) y el uso de atributos válidos. Al seguir estas pautas, se mejora la interoperabilidad y la experiencia del usuario, además de contribuir a un ecosistema web más accesible.
    2. Reglas para escribir HTML: Al escribir HTML, es fundamental seguir ciertas reglas para garantizar que el código sea eficiente y semántico. Entre las más importantes se incluye la correcta estructuración del documento, que debe contener las etiquetas esenciales como "html", "head", y "body". Además, se deben utilizar etiquetas semánticas como "header", "footer", y "article" para mejorar la accesibilidad y optimizar el SEO. También es crucial asegurarse de cerrar todas las etiquetas que lo requieran, utilizar atributos válidos y mantener un formato limpio con buena indentación, lo que facilita la legibilidad y el mantenimiento del código.
    3. Estructura básica de un archivo web: La estructura básica de un archivo HTML es fundamental para el desarrollo de una página web. Un archivo típico comienza con la declaración del tipo de documento (DOCTYPE html), seguida de la etiqueta "html" que encierra todo el contenido. Dentro del "head", se incluyen metadatos importantes como el conjunto de caracteres y el título de la página, además de enlaces a hojas de estilo externas. El "body" contiene el contenido visible para el usuario, organizado en secciones mediante etiquetas como "header", "main", y "footer", lo que facilita la navegación y la comprensión del contenido presentado.

    1. Etiquetas (nombre al menos 40): html, head, title, meta, link, script, style, body, header, footer, main, section, article, nav, h1, h2, h3, h4, h5, h6, p, a, img, ul, ol, li, table, tr, td, th, caption, form, input, textarea, button, select, option, label, div, span
    2. Atributos: Los atributos en HTML son propiedades que se pueden añadir a las etiquetas para proporcionar información adicional sobre un elemento. Se colocan dentro de la etiqueta de apertura y suelen estar compuestos por un nombre y un valor, separados por un signo igual.
    3. Elemento HTML: Un elemento HTML es la unidad básica del contenido en un documento web. Se compone de una etiqueta de apertura, contenido y una etiqueta de cierre. Estos elementos estructuran y organizan la información en la página, permitiendo a los navegadores interpretarla y mostrarla correctamente.
    4. Elementos en línea y en bloque: Los elementos HTML se clasifican en dos categorías principales: elementos en bloque y elementos en línea. Los elementos en bloque, como "div", "p", y "h1", ocupan todo el ancho disponible y siempre comienzan en una nueva línea, lo que los hace ideales para estructurar el contenido en secciones bien definidas. Por otro lado, los elementos en línea, como "span", "a", y "img", solo ocupan el ancho necesario y no inician una nueva línea, permitiendo que se integren dentro del flujo de texto. Esta clasificación es crucial para el diseño y la presentación del contenido en la web, ya que determina cómo se visualiza y organiza la información.
